## 一、产品概述

### 1.1 产品定位

**传奇剧场**是一款面向企业家的 AI 短剧剧本生成平台。用户通过 15 道递进式问答讲述真实创业经历，AI 自动将其重构为专业级短剧剧本，支持三种创作方向和三档集数选择。

**一句话**: 把你的创业经历，变成一部专属于你的短剧剧本。

### 1.2 核心价值

| 维度 | 传统编剧 | 传奇剧场 |
|------|----------|----------|
| 时间 | 3-6 个月 | 15 分钟问答 + 60 秒生成 |
| 成本 | 1-10 万元/集 | 免费体验 / ¥4,980-19,800 |
| 门槛 | 需要编剧资源 | 零写作能力要求 |
| 个性化 | 编剧理解偏差 | AI 基于本人真实经历 |

### 1.3 目标用户

- **核心**: 有故事的创业者/企业家（35-55 岁男性为主）
- **扩展**: 品牌主理人、投资人、企业高管

## 二、用户流程

### 2.1 主流程

```
落地页 → 选择创作方向 → 回答 15 题 → AI 生成 → 结果展示 → 导出/升级
```

### 2.2 详细步骤

| 步骤 | 页面 | 用户操作 | 耗时 |
|------|------|----------|------|
| 1 | 落地页 | 浏览产品价值 + 定价方案，点击"开始创作" | 30s |
| 2 | 方向选择 | 三选一：👑传奇之路 / 🏛️品牌之路 / 🔥狂暴之路 | 10s |
| 3 | 问答 | 回答 15 道递进式问题（4 层：起点→转折→情感→精神） | 15min |
| 4 | 生成中 | 等待 AI 生成（含进度动画） | 30-60s |
| 5 | 结果页 | 7 个 Tab 浏览剧本 + 导出 PDF/DOCX + 查看升级方案 | 不限 |

### 2.3 三步指示器

所有页面顶部显示进度：
```
● 创作方向 ──→ ○ 回答问题 ──→ ○ 生成剧本
```

## 三、三方向体系

### 3.1 传奇之路（👑 个人传记）

**定位**: 忠实还原创业经历，强化戏剧冲突，打造企业家个人 IP

**15 题结构**:
| 层级 | 题目数 | 主题 |
|------|--------|------|
| 第 1 层 · 起点 | 5 题 | 创业前状态、契机、团队、第一版产品、第一次正反馈 |
| 第 2 层 · 转折 | 5 题 | 最大困难、关键帮助、转折点、最冒险决定、放弃念头 |
| 第 3 层 · 情感 | 4 题 | 感动瞬间、想感谢的人、失去与值得、重来选择 |
| 第 4 层 · 精神 | 1 题 | 核心精神、对创业者的话 |

**AI 指令侧重**: 对标《中国合伙人》《社交网络》叙事品质，商业细节真实可信，用户提到的真实人物和事件必须尊重。

### 3.2 品牌之路（🏛️ 品牌创意叙事）

**定位**: 将品牌核心卖点置入创意设定（穿越/重生/平行世界），用高概念叙事传播品牌

**15 题结构**:
| 层级 | 题目数 | 主题 |
|------|--------|------|
| 第 1 层 · 品牌基因 | 5 题 | 品牌名、解决痛点、核心卖点、客户案例、传播目标 |
| 第 2 层 · 创意设定 | 5 题 | 时代背景、核心冲突、反派设定、主角身份、获胜方式 |
| 第 3 层 · 叙事风格 | 4 题 | 调性、真实案例改编、感情线、品牌露出方式 |
| 第 4 层 · 传播目标 | 1 题 | 投放平台、目标受众 |

**AI 指令侧重**: 对标《我在古代开星巴克》，品牌作为故事核心驱动，结尾呼应现实品牌形成惊喜。

### 3.3 狂暴之路（🔥 脑洞爽片）

**定位**: 企业家本人出演，短剧算法驱动，高冲突强反转，最大化传播效果

**15 题结构**:
| 层级 | 题目数 | 主题 |
|------|--------|------|
| 第 1 层 · 人设打造 | 5 题 | 剧中人设、核心特质、口头禅、搭档设定、想虐的人 |
| 第 2 层 · 爽点设计 | 5 题 | 爽点类型、炸裂场景、反转时机、传播台词、隐藏身份 |
| 第 3 层 · 出演意愿 | 4 题 | 出镜程度、银幕形象、讨厌套路、终极目标 |
| 第 4 层 · 狂野设定 | 1 题 | 无限制狂野设定 |

**AI 指令侧重**: 对标短剧爆款算法，前 3 秒强冲突、每 90 秒爽点、金句密度翻倍，标注用户亲自出镜的高光时刻。

## 四、技术架构

### 4.1 技术栈

| 层 | 技术 | 说明 |
|----|------|------|
| 前端 | React 19 + Vite 8 | SPA，无路由（Zustand 状态驱动切换） |
| 样式 | TailwindCSS v4 + 自定义 CSS | 暗金主题，CSS 变量管理 |
| 状态管理 | Zustand | step-based 页面路由 + answers 存储 |
| 后端 | Express.js (Node.js) | 单一 API 服务，端口 3001 |

## 五、API 设计

### 5.1 POST /api/generate

**请求**:
```json
{
  "answers": { "1": "2015年在一家AI公司...", ... },
  "styleName": "企业家励志型",
  "scriptCount": 30,
  "direction": "legend"
}
```

**响应**:
```json
{
  "oneLine": "一句话故事（20字内）",
  "synopsis": "300字故事简介",
  "characters": {
    "protagonist": { "name": "", "age": "", "traits": "", "arc": "" },
    "supporting": [{ "name": "", "role": "", "traits": "", "relation": "", "function": "" }],
    "antagonist": { "description": "" }
  },
  "episodes": [{ "ep": 1, "title": "", "summary": "", "hook": false, "emotion": "燃" }],
  "scriptSample": { "episode": 1, "title": "", "scenes": [{ "location": "", "time": "", "characters": [], "emotion": "", "content": "" }] },
  "evaluation": { "goldOpening": 9, "rhythmDesign": 8, ... },
  "commercial": { "platforms": [], "payNodes": [], "competitors": [], "goldenLines": [] }
}
```

### 5.2 GET /api/health

```json
{ "status": "ok", "model": "deepseek-v4-flash", "apiConfigured": true }
```

### 5.3 JSON 解析策略（4 层兜底）

| 层 | 策略 | 说明 |
|----|------|------|
| 1 | jsonrepair 直接修复 | 自动处理常见 JSON 语法错误 |
| 2 | 去尾逗号 + jsonrepair | 修复对象/数组末尾多余的逗号 |
| 3 | 转义换行符 + jsonrepair | 修复字符串内部未转义的 `\n` `\t` |
| 4 | 逐字段正则提取 | 用正则分别提取 oneLine/synopsis/characters 等字段 |

### 5.4 Prompt 结构

```
System Prompt（约 3500 tokens）:
├── 角色设定（花火智作首席编剧 AI）
├── 核心创作理念（5 条标准）
├── 2026年热点趋势（8 条）
├── 风格要求
├── 方向专属指令（legend/brand/fury）
├── 八维质量评估标准
├── JSON 输出格式规范
├── 严格创作原则（9 条）
└── 格式铁律（JSON 转义规则）

User Prompt:
├── 风格适配标签
└── 用户 15 题回答原文
```

## 六、八维评估体系

| 维度 | 权重 | 检查点 |
|------|------|--------|
| 黄金开场 | **1.5** | 前 100 字是否有冲突/反差/悬念 |
| 悬念钩子 | **1.4** | 每集末 3 句是否制造强烈悬念 |
| 节奏设计 | **1.3** | 每 30 秒微转折，爽点均匀分布 |
| 情绪价值 | **1.2** | 情绪引爆点有无充分铺垫 |
| 台词质量 | **1.1** | 每句是否有信息增量，金句密度 |
| 大纲结构 | 1.0 | 奠基→成长→危机→逆袭→升华 |
| 人物成长 | 1.0 | 主角从 A 到 B 的转变可信度 |
| 逻辑纠错 | 0.8 | 时间线/因果/商业逻辑 |

**评级**:
- S+（70-80）: 可直接拍摄上线
- A（60-69）: 微调后可上线
- B（50-59）: 需较大修改
- C（<50）: 建议重构

## 八、UI 设计规范

### 8.1 色彩系统

| 变量 | 色值 | 用途 |
|------|------|------|
| 主背景 | `#0a0a0a` | 全页面底色 |
| 卡片背景 | `#141414` | 卡片/面板 |
| 输入框背景 | `#1a1a1a` | 表单输入 |
| 主金色 | `#c8953b` | 标题高亮、CTA 按钮、评分 |
| 金浅色 | `#d4a853` | 渐变、悬停 |
| 主文字 | `#ede8dd` | 正文、标题 |
| 次要文字 | `#9a9488` | 说明文字 |
| 弱化文字 | `#5c564a` | 占位、辅助信息 |
| 边框 | `#2a2518` | 卡片边框 |

### 8.2 设计原则

- 暗黑底色 + 暖金点缀 → 企业家气质的厚重与精致
- 大字号、宽间距、克制留白
- 卡片圆角 12px，按钮 8px
- 主按钮金色渐变，悬停提升 + 发光
- 金色流光文字（`text-shimmer`）用于主标题"传奇"

### 8.3 响应式

- 完整支持移动端（375px+）和桌面端
- 定价网格：移动 1 列 → 768px+ 4 列
- 四步流程：移动 2 列 → 768px+ 4 列
